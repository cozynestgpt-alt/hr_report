# 월례회의 인원보고서 - 팀원 실행 매뉴얼

NAS 경로: `\\nas\CN_AMD\월례회의_자료\hr_report`

## 1. 실행 방법

1. 위 NAS 경로로 이동해서 **`월례회의_인원보고_실행.bat`** 파일을 더블클릭합니다.
2. 검정 화면(터미널)이 뜨고 아래처럼 기준월을 입력하라는 문구가 나옵니다.
   ```
   기준월을 입력하세요 (예: 202605, 엔터만 누르면 당월 자동 적용):
   ```
   - 보고 대상 월을 `YYYYMM` 형식(예: `202605`)으로 입력하고 엔터
   - 그냥 엔터만 치면 이번 달 기준으로 자동 실행됩니다
3. 처리 진행 상황이 화면에 순서대로 출력됩니다(사원명단 로드 → 인원비교보고서 집계 → 시트 채우기 → 저장 → 요약 작성).
4. "작업이 완료되었습니다" 메시지가 뜨면 **`output` 폴더**에서 결과물을 확인합니다.
   - `월례회의_인원보고_YYYYMM.xlsx` → 보고용 엑셀
   - `인원변동요약_YYYYMM.txt` → 전년 대비 증감 요약 텍스트
5. 화면 하단에 안내된 대로 **엔터를 누르면 창이 닫힙니다.**

## 2. 코드 업데이트 방법 (담당자/IT용)

팀원 대부분은 이 단계를 직접 할 필요가 없습니다. NAS의 코드가 오래되었거나 담당자가 새 기능을 배포했다고 안내한 경우에만 아래를 수행하세요.

1. NAS 폴더(`\\nas\CN_AMD\월례회의_자료\hr_report`)에서 명령 프롬프트(cmd)를 엽니다.
2. 아래 명령 실행:
   ```bash
   git pull
   ```
3. `Already up to date.`가 아니라 파일이 갱신되는 로그가 뜨면 정상적으로 최신 코드를 받은 것입니다.

> `input/`, `output/`, `DB/`, `참고/` 폴더의 실데이터 파일은 git에 포함되지 않으므로 `git pull`로 지워지거나 바뀌지 않습니다.

## 3. 자주 발생하는 오류 및 해결 방법

### "input 폴더에 '사원명단*.xlsx' 파일이 없습니다"
- `input` 폴더에 최신 사원명단·사원현황 xlsx 파일이 없는 경우 발생합니다.
- 담당자에게 최신 파일을 `input` 폴더에 넣어달라고 요청하세요.

### "Python이 설치되어 있지 않습니다"
- 실행 PC에 Python이 설치되어 있지 않거나 PATH에 등록되지 않은 경우입니다.
- IT 담당자에게 Python 설치를 요청하세요.

### "폴더에 접속할 수 없습니다" / bat 실행 시 바로 오류창
- NAS 드라이브 연결이 끊겼거나 네트워크 문제일 수 있습니다.
- 탐색기에서 `\\nas\CN_AMD\월례회의_자료\hr_report` 경로가 정상적으로 열리는지 먼저 확인하세요.

### 기준월 형식 오류 ("YYYYMM 형태로 입력하세요")
- 기준월은 `202605`처럼 6자리 숫자(연도 4자리 + 월 2자리)로만 입력해야 합니다.

### output 파일이 기존 것과 그대로 덮어써짐
- 같은 월을 다시 실행하면 기존 결과 파일이 자동으로 덮어써집니다. 정상 동작입니다.
- 이전 결과를 보존하고 싶다면 재실행 전에 `output` 폴더의 파일을 다른 곳에 백업해두세요.

### 그 외 오류
- 화면에 출력된 오류 메시지 전체를 캡처해서 담당자에게 전달해주세요.
